CUT MAPPING TABLE (SESSION REFERENCE)

Purpose
- Reference table for launch cut behavior using ROM offsets 0x6125 and 0x6126.
- Mapping below reflects the current interpretation validated during this session.

Offsets
- Byte A: 0x6125
- Byte B: 0x6126

Definition (session)
- Cut pair (0x6125 / 0x6126) = fuel cut mapping reference.

State Mapping (current)
- 00 / 00 = fuel cut active + ign cut inactive
- FF / FF = fuel cut inactive + ign cut active
- FF / 00 = fuel cut active + ign cut active

VTEC NOVTEC Mapping Notes
- p28.bin vs p28novtec.bin and p30.bin vs p30novtec.bin both differ by the same 4 bytes:
	- 0x6120: 0x44 -> 0x43
	- 0x61F2: 0xFF -> 0x00
	- 0x6657: 0x52 -> 0x54
	- 0x6659: 0x37 -> 0x33
- Common "wiki" VTEC offsets checked during comparison remained unchanged in these custom AF 22 B6 bins.

How to identify the real VTEC ON/OFF address (safe method)
- Do not infer from multi-byte patch alone; isolate one byte at a time.
- Build 4 test bins from the same base and modify only one candidate byte per bin:
	- Test A: 0x6120 only
	- Test B: 0x61F2 only
	- Test C: 0x6657 only
	- Test D: 0x6659 only
- Flash one test at a time, then run same driving condition and monitor VTEC status.
- The byte that consistently disables VTEC by itself is the primary VTEC switch for this ROM family.
- Current working hypothesis: 0x61F2 is the strongest candidate.

Fuel Map Zones (session definitions)

	- Full table range: 0x6EAD..0x707E
	- Geometry: 20 rows x 10 columns
	- Row stride: 0x18
	- Index formula:
		- offset = 0x6EAD + row * 0x18 + col
		- row = 0..19
		- col = 0..9
	- Confirmed corners:
		- top-left = 0x6EAD
		- top-right = 0x6EB6
		- bottom-left = 0x7075

		- p28+5lowcam1erverhorizon.bin -> first horizontal row = 0x6EAD..0x6EB6

FAN CONTROL Mapping Notes (validated)
- Activation byte offset:

	- 0x00 = fan control disabled
	- Validation:

- Fan temperature value byte offset:
	- 0x616E
	- This byte stores the fan threshold value used by this ROM family.
	- Validation with available variants:
		- p28.bin: 0x1F
		- p28fancontrol80.bin: 0x1F (identical to p28.bin)
		- p28fancontrol87.bin: 0x22

- Quick rule for editing:
	- Toggle fan control using 0x616D.
	- Adjust fan threshold value using 0x616E.
